## 第三章 行为准则

### 3.1 核心价值观

微波室全体成员应共同遵守以下核心价值观：

#### 价值观一：客户导向

**定义**：以客户需求为中心，深入理解客户需求，快速响应客户反馈。

**行为要求（Do）**：
- 主动了解客户的真实需求和痛点
- 站在客户角度思考问题，提供最优解决方案
- 对客户反馈快速响应，及时处理
- 定期与客户沟通，保持良好关系
- 客户满意度作为工作的重要衡量标准

**禁止行为（Don't）**：
- 忽视客户需求，闭门造车
- 对客户需求敷衍了事，推诿扯皮
- 隐瞒问题，不主动沟通
- 过度承诺，无法兑现

#### 价值观二：质量第一

**定义**：质量是产品的生命线，严守质量标准，不妥协、不侥幸。

**行为要求（Do）**：
- 严格遵守设计规范和测试标准
- 设计评审前自查，确保文档完整、逻辑正确
- 发现问题及时报告，不隐瞒、不拖延
- 测试充分，确保产品满足规格要求
- 持续改进，追求零缺陷

**禁止行为（Don't）**：
- 为了进度牺牲质量
- 隐瞒缺陷或数据造假
- 侥幸心理，认为"应该没问题"
- 不按照流程执行，偷工减料

#### 价值观三：协同合作

**定义**：团队成功大于个人成功，主动沟通，乐于分享，互相支持。

**行为要求（Do）**：
- 主动与上下游沟通，确保信息同步
- 遇到问题及时求助，也乐于帮助他人
- 知识经验主动分享，不做"知识守财奴"
- 跨组协作积极配合，以大局为重
- 尊重他人意见，理性沟通，求同存异

**禁止行为（Don't）**：
- 信息孤岛，不与他人分享
- 遇到问题不沟通，自己硬扛
- 恶意不配合，故意设置障碍
- 破坏团队氛围，传播负能量

#### 价值观四：持续改进

**定义**：追求卓越的永动力，主动学习，勇于创新，持续优化。

**行为要求（Do）**：
- 保持学习热情，跟踪技术发展趋势
- 定期复盘总结，从成功和失败中学习
- 勇于尝试新方法、新技术
- 对现有流程和工具提出改进建议
- 每周参加技术分享，主动分享经验

**禁止行为（Don't）**：
- 固步自封，拒绝学习新知识
- 重复犯同样的错误，不总结教训
- 因循守旧，不愿改变
- 对改进建议嗤之以鼻

#### 价值观五：诚信正直

**定义**：做人做事的底线，实事求是，勇于担责，信守承诺。

**行为要求（Do）**：
- 实事求是，不夸大、不隐瞒
- 勇于承认错误，承担相应责任
- 信守承诺，说到做到
- 廉洁自律，不谋取不正当利益
- 对事不对人，客观公正

**禁止行为（Don't）**：
- 虚报瞒报，欺骗领导和同事
- 推卸责任，找借口甩锅
- 言而无信，承诺了不兑现
- 利用职务之便谋取私利

### 3.2 行为规范总则

除上述核心价值观外，所有成员还应遵守：

1. **遵守规章制度**：遵守公司制度和微波室各项规定
2. **保守商业秘密**：严守公司技术机密和商业秘密，不对外泄露
3. **尊重知识产权**：尊重他人知识产权，职务发明及时申请专利
4. **维护公司利益**：不做损害公司利益的事情
5. **保持良好职业操守**：诚实守信，廉洁自律

## 第四章 工作制度

### 4.1 考勤管理

#### 工作时间

- **标准工作时间**：周一至周五 9:00-18:00
- **弹性工作时间**：可弹性±1小时（即8:00-17:00 或 10:00-19:00）
- **午休时间**：12:00-13:00
- **核心工作时间**：10:00-16:00（确保此时间段内在岗，便于协作）

#### 考勤规定

- **迟到/早退**：每月累计不超过3次，每次不超过30分钟
- **迟到/早退处理**：超过规定次数的，按公司规定处理
- **旷工**：无故缺勤视为旷工，按公司规定严肃处理
- **考勤记录**：通过OA系统打卡，作为考勤依据

#### 请假制度

**请假类型**：
- 事假：因私事请假
- 病假：因病请假，需提供医院证明
- 年假：按工龄享受的带薪年假
- 婚假/产假/丧假：按国家规定执行

**请假流程**：
1. 提前在OA系统提交请假申请
2. 请假1天内：组长审批
3. 请假1-3天：组长审批→副主任审批
4. 请假3天以上：组长审批→副主任审批→主任审批
5. 紧急情况可先口头请假，事后补办手续

#### 加班管理

- **加班申请**：加班需提前申请，经组长批准
- **加班补偿**：优先安排调休，无法调休的按公司规定发放加班费
- **审批权限**：组长审批，每月汇总报主任

### 5.4 变更管理规范

#### 变更分级

| 变更级别 | 定义 | 审批权限 | 处理时效 |
|:---------|:-----|:---------|:---------|
| **一级变更** | 影响项目目标、成本、进度超过10% | 主任 | 3个工作日内决策 |
| **二级变更** | 影响功能、性能、接口 | 项目经理 | 2个工作日内决策 |
| **三级变更** | 文档更新、优化改进，不影响功能和进度 | 技术负责人 | 1个工作日内决策 |

#### 变更流程

```
① 变更申请 → ② 影响分析 → ③ 变更审批 → ④ 变更实施 → ⑤ 变更验证 → ⑥ 变更关闭
```

**详细步骤**：

1. **变更申请**：填写变更请求表（CR），说明变更原因和内容
2. **影响分析**：评估变更对需求、设计、进度、成本的影响
3. **变更审批**：按级别提交相应审批人审批
4. **变更实施**：审批通过后执行变更，更新相关文档
5. **变更验证**：验证变更效果，确保满足要求
6. **变更关闭**：归档变更记录，关闭变更请求

#### 变更请求表（CR）要素

| 要素 | 说明 |
|:-----|:-----|
| 变更编号 | CR-YYYY-NNN |
| 变更类型 | 需求变更/设计变更/计划变更/其他 |
| 变更原因 | 为什么需要变更 |
| 变更内容 | 具体变更什么 |
| 影响分析 | 对需求/设计/进度/成本/质量的影响 |
| 审批意见 | 批准/拒绝/延后 |
| 实施计划 | 如何实施、谁负责、什么时候完成 |

### 5.5 组间协作规范

#### 产品开发组 ↔ 技术开发组协作流程

```
① 需求提出 → ② 需求评估 → ③ 方案制定 → ④ 开发实现 → ⑤ 验收交付 → ⑥ 技术支持
```

**详细步骤**：

1. **需求提出**（产品开发组）
   - 产品开发组向技术开发组组长提出模块需求
   - 明确性能指标、接口要求、交付时间

2. **需求评估**（技术开发组）
   - 技术开发组组长组织评估
   - 评估可行性、工作量、风险
   - 2个工作日内反馈评估结果

3. **方案制定**（技术开发组）
   - 制定技术方案
   - 组织方案评审（产品开发组参与）
   - 5个工作日内完成方案

4. **开发实现**（技术开发组）
   - 按计划开发模块
   - 定期汇报进度
   - 发现风险及时反馈

5. **验收交付**（双方）
   - 产品开发组验收模块
   - 确认满足需求后签收
   - 交付设计文档和测试报告

6. **技术支持**（技术开发组）
   - 提供技术支持
   - 解决使用过程中的问题
   - 根据反馈优化模块

#### 协作时效要求

| 环节 | 时效要求 |
|:-----|:---------|
| 需求评估反馈 | 2个工作日内 |
| 技术方案制定 | 5个工作日内 |
| 进度风险反馈 | 发现后立即反馈 |
| 问题响应 | 4小时内响应，2个工作日内给出解决方案 |

#### 协作沟通方式

- **日常工作对接**：产品开发组 → 技术开发组组长 → 各专业工程师
- **定期协调**：每周三下午周协调会
- **紧急事项**：电话或即时通讯，必要时召集临时会议
- **重大事项**：通过主任/副主任协调

### 7.3 奖惩流程

#### 奖励流程

```
① 提名/申请 → ② 材料审核 → ③ 评审委员会评审 → ④ 公示 → ⑤ 颁发奖励
```

**详细步骤**：

1. **提名/申请**：个人申请或上级提名，提交申请材料
2. **材料审核**：综合管理组审核材料完整性和真实性
3. **评审委员会评审**：由主任、副主任、组长组成评审委员会评审
4. **公示**：评审结果公示3个工作日
5. **颁发奖励**：公示无异议后颁发奖励

#### 惩戒流程

```
① 发现问题 → ② 调查核实 → ③ 听取申辩 → ④ 做出决定 → ⑤ 执行并记录
```

**详细步骤**：

1. **发现问题**：上级发现或他人举报
2. **调查核实**：综合管理组调查核实，收集证据
3. **听取申辩**：听取当事人申辩，必要时组织听证会
4. **做出决定**：根据调查结果做出处理决定
5. **执行并记录**：执行惩戒措施，记入档案

**申诉权利**：当事人对处理结果有异议，可在5个工作日内向副主任/主任申诉

## 第八章 沟通与反馈

### 8.1 沟通渠道

| 渠道 | 用途 | 联系方式 | 响应时效 |
|:-----|:-----|:---------|:---------|
| **直属组长** | 日常工作安排、绩效反馈、发展建议 | 面对面/OA/邮件 | 2个工作日内 |
| **组长** | 组内事务协调、资源申请、问题升级 | 面对面/OA/邮件 | 2个工作日内 |
| **副主任** | 重大事项、跨组协调、申诉 | 邮件/面谈 | 3个工作日内 |
| **主任** | 战略事项、重大决策、申诉 | 邮件/面谈 | 5个工作日内 |
| **综合管理** | 制度咨询、后勤事务、员工关系 | OA/邮件 | 2个工作日内 |

**沟通原则**：
- 日常工作先找直属组长
- 跨组事项通过组长协调
- 重大事项抄送副主任/主任
- 紧急情况可越级汇报

### 8.2 反馈机制

#### 定期面谈

| 面谈类型 | 时间 | 参与人 | 内容 |
|:---------|:-----|:-------|:-----|
| **试用期面谈** | 入职1个月、3个月 | 组长+新员工 | 适应情况、工作反馈、改进建议 |
| **绩效面谈** | 每季度末 | 组长+员工 | 绩效回顾、优点不足、改进方向 |
| **发展面谈** | 每年末 | 组长+员工 | 职业发展、培养计划、期望沟通 |

**面谈要求**：
- 提前准备，有明确的议题
- 双向沟通，听取员工想法
- 形成结论，明确行动计划
- 记录存档，跟踪落实

#### 满意度调查

| 调查类型 | 频率 | 方式 | 内容 |
|:---------|:-----|:-----|:-----|
| **季度团队满意度** | 每季度 | 匿名问卷 | 团队协作、工作氛围、管理支持 |
| **年度员工满意度** | 每年 | 问卷+访谈 | 全面评估，包括薪酬、发展、管理等方面 |

**调查结果应用**：
- 分析共性问题，制定改进措施
- 识别优秀实践，推广复制
- 作为管理改进的输入

#### 意见箱/热线

- **意见箱**：设在公共区域，匿名投递
- **意见邮箱**：综合管理的邮箱，可匿名发送
- **热线**：综合管理内线，工作时间可拨打

**处理流程**：
- 每周收集整理意见
- 分类处理，能立即回复的立即回复
- 需要调查的研究后回复
- 重大意见报主任处理
- 处理结果反馈给提意见人（如非匿名）

### 8.3 申诉机制

#### 申诉范围

- 对绩效考核结果有异议
- 对奖惩处理结果有异议
- 对工作岗位安排有异议
- 其他认为不公正的待遇

#### 申诉流程

```
① 提出申诉 → ② 受理审查 → ③ 调查核实 → ④ 做出裁决 → ⑤ 反馈结果
```

**详细步骤**：

1. **提出申诉**：书面提交申诉材料，说明申诉理由和诉求
2. **受理审查**：5个工作日内决定是否受理，并通知申诉人
3. **调查核实**：成立调查组，调查核实情况
4. **做出裁决**：根据调查结果做出裁决
5. **反馈结果**：5个工作日内反馈裁决结果

**申诉时效**：
- 应在知道或应当知道相关事项后10个工作日内提出
- 特殊情况可适当延长，但最长不超过30个工作日

**申诉层级**：
- 对组长处理结果有异议 → 向副主任申诉
- 对副主任处理结果有异议 → 向主任申诉
- 对主任处理结果有异议 → 向公司人力资源部门申诉

**申诉原则**：
- 实事求是，不得捏造事实
- 不影响正常工作
- 不影响对申诉人的正常管理
