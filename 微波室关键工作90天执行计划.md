# 微波室关键工作执行计划（90天）

## 必须开展的核心工作

**目标**：3个月内建立基础运转机制，让25人团队能够高效协同、持续学习

---

## 一、5项关键工作概览

| 序号 | 关键工作 | 为什么必须 | 预期效果 | 负责人 | 完成时间 |
|:----:|:---------|:-----------|:---------|:------:|:--------:|
| **1** | **建立组间协作机制** | 产品开发组4人 ↔ 技术开发组11人协作不畅是最大痛点 | 需求对接顺畅，模块复用率提升 | 主任 | 第1周 |
| **2** | **启动周技术分享** | 25人知识孤岛严重，必须快速打破 | 知识流动，团队能力共同提升 | 主任 | 第1周 |
| **3** | **建立需求管理流程** | 需求不清导致返工是最大浪费 | 需求明确，减少返工 | 产品管理组长 | 第2周 |
| **4** | **试运行绩效管理** | 没有考核标准无法激励和区分 | 奖优罚劣，激发动力 | 主任 | 第4周 |
| **5** | **启动IPD试点项目** | 流程必须在实践中验证和优化 | 流程跑通，形成标杆 | 主任 | 第4周 |

## 二、详细执行计划

### 关键工作1：建立组间协作机制

**目的**：打通产品开发组与技术开发组的协作通道

**核心动作**：

| 时间 | 动作 | 交付物 | 参与人 |
|:----:|:-----|:-------|:-------|
| **第1天** | 宣布建立"模块产品经理"制度 | 通知邮件 | 全员 |
| **第3天** | 确定4个模块产品经理（天线/射频/数字/电源各1人） | 对接人名单 | 技术开发组长 |
| **第5天** | 召开第一次周协调会 | 会议纪要 | 产品开发组长+技术开发组长+4模块产品经理 |
| **每周三下午** | 固定召开周协调会（持续） | 会议纪要+行动项 | 同上 |

**周协调会标准议程**（30分钟）：
```
1. 产品开发组需求更新（5分钟）- 新需求、变更需求
2. 技术开发组进度汇报（10分钟）- 各模块进展
3. 问题协调（10分钟）- 需求不明确、进度风险、技术问题
4. 下周计划确认（5分钟）- 交付物、里程碑
```

**你的参与**：每月参加1次，了解协作情况，解决重大问题

---

### 关键工作2：启动周技术分享

**目的**：建立学习型组织文化，打破知识孤岛

**核心动作**：

| 时间 | 动作 | 交付物 | 参与人 |
|:----:|:-----|:-------|:-------|
| **第1天** | 宣布每周五下午为"学习时间"，不安排其他会议 | 通知邮件 | 全员 |
| **第1天** | 制定分享排班表（先排4周） | 排班表 | 综合管理 |
| **第1周五** | 你亲自做第一次分享（主题：微波室技术愿景） | 分享PPT+视频 | 全员 |
| **每周五下午** | 固定技术分享（持续） | 分享PPT+知识库文档 | 全员 |

**分享排班表（前4周示例）**：

| 周次 | 分享组 | 主题方向 | 分享人 | 审核人 |
|:----:|:-------|:---------|:-------|:-------|
| 第1周 | 主任 | 微波室技术愿景与发展规划 | 你 | - |
| 第2周 | 技术开发-天线 | 某型天线设计经验分享 | XXX | 你 |
| 第3周 | 技术开发-射频 | 低噪声放大器设计技巧 | XXX | 技术开发组长 |
| 第4周 | 产品开发 | 项目复盘与经验总结 | XXX | 产品开发组长 |

**分享要求**：
- 分享前3天提交PPT给你或组长审核
- 分享30分钟 + 讨论30分钟
- 分享后1天内上传文档到知识库
- 每季度评选"最佳分享奖"（奖金500元）

---

### 关键工作3：建立需求管理流程

**目的**：让需求清晰可追溯，减少返工浪费

**核心动作**：

| 时间 | 动作 | 交付物 | 负责人 |
|:----:|:-----|:-------|:-------|
| **第8天** | 建立需求台账模板 | Excel模板 | 产品管理组长 |
| **第10天** | 培训需求台账使用方法 | 培训记录 | 产品管理组长 |
| **第14天** | 梳理现有项目需求，录入台账 | 需求台账V1.0 | 产品管理组 |
| **第14天** | 制定《需求分析检查表》 | 检查表文档 | 产品管理组长 |
| **持续** | 每个新项目必须填需求台账 | 更新台账 | 产品管理组 |

**需求台账字段**：
| 字段 | 说明 |
|------|------|
| 需求编号 | REQ-YYYY-NNN |
| 需求名称 | 简明描述 |
| 来源 | 客户名称/内部 |
| 类型 | 探测类/通信类 |
| 提出日期 | 日期 |
| 责任人 | 产品经理 |
| 优先级 | P0/P1/P2/P3 |
| 状态 | 待分析/分析中/已确认/已实现/已关闭 |
| 期望交付日期 | 日期 |
| 实际交付日期 | 日期 |

**需求分析检查表**：
- [ ] 需求是否明确可验证？
- [ ] 技术可行性是否评估？
- [ ] 对现有模块的影响是否分析？
- [ ] 交付时间和资源需求是否明确？
- [ ] 相关方是否确认？

### 关键工作4：试运行绩效管理

**目的**：建立评价标准，奖优罚劣，激发团队动力

**核心动作**：

| 时间 | 动作 | 交付物 | 负责人 |
|:----:|:-----|:-------|:-------|
| **第15天** | 确定三组考核维度和权重 | 绩效考核方案V1.0 | 你 |
| **第20天** | 设计绩效目标表模板 | 模板文档 | 综合管理 |
| **第22天** | 培训绩效管理办法 | 培训记录 | 你 |
| **第25天** | 各组长与成员签订Q1绩效目标 | 绩效目标表 | 各组长 |
| **每月** | 月度绩效跟踪反馈 | 反馈记录 | 各组长 |
| **第90天** | Q1绩效考核评估 | 考核结果 | 各组长+你 |

**三组考核维度（简化版）**：

| 组 | 考核维度 | 权重 | 关键指标（示例） |
|:--:|:---------|:----:|:-----------------|
| **技术开发** | 技术成果 | 40% | 模块按时交付率、性能达标率 |
| | 项目支持 | 30% | 需求响应及时率、支持满意度 |
| | 协作配合 | 30% | 协作满意度、文档贡献 |
| **产品开发** | 产品成果 | 50% | 产品按时交付率、一次通过率 |
| | 跨组协作 | 30% | 需求明确度、协作满意度 |
| | 技术牵引 | 20% | 参与技术规划次数 |
| **产品管理** | 项目成功 | 40% | 项目按时交付率、客户满意度 |
| | DFX质量 | 35% | 评估准确率、问题发现率 |
| | 协调能力 | 25% | 问题响应时间、流程执行 |

**绩效目标表示例**：
```
员工：张三        岗位：天线工程师        考核周期：Q1 2024

目标1：完成XX通信天线模块设计
- 衡量标准：性能指标达成率≥95%，按时交付
- 权重：40%

目标2：支持产品开发组Y项目天线需求
- 衡量标准：需求响应及时率100%，支持满意度≥4.0
- 权重：30%

目标3：提交技术文档
- 衡量标准：提交天线设计规范文档2篇
- 权重：20%

目标4：参与技术分享
- 衡量标准：完成1次组内技术分享
- 权重：10%
```

### 关键工作5：启动IPD试点项目

**目的**：在实战中验证和优化IPD流程

**核心动作**：

| 时间 | 动作 | 交付物 | 负责人 |
|:----:|:-----|:-------|:-------|
| **第22天** | 选定试点项目（中等复杂度，涉及多模块） | 试点项目确定 | 你 |
| **第25天** | 组建试点项目团队（含各组代表） | 项目团队名单 | 你 |
| **第28天** | 项目启动会，明确目标和计划 | 会议纪要 | 项目经理 |
| **第30天** | 完成概念阶段，召开CDCP评审 | 评审报告 | 你主持 |
| **第45天** | 完成计划阶段，召开PDCP评审 | 评审报告 | 你主持 |
| **第60天** | 开发阶段中期检查 | 检查报告 | 项目经理 |
| **第75天** | 完成开发阶段，召开TR评审 | 评审报告 | 你参与 |
| **第90天** | 试点项目复盘总结 | 复盘报告 | 项目经理 |

**试点项目选择标准**：
- [ ] 中等复杂度（2-3个月可完成）
- [ ] 涉及多个技术模块（天线+射频+数字+电源）
- [ ] 有明确的产品交付目标
- [ ] 团队成员有代表性（各组都有人参与）

**关键评审点你必须参加**：
- CDCP（概念决策评审）：评估需求和技术可行性
- PDCP（计划决策评审）：评估方案和资源配置
- ADCP（发布决策评审）：评估质量和交付 readiness

## 五、关键成功因素

| 因素 | 具体做法 |
|:-----|:---------|
| **领导带头** | 你亲自参加周协调会（每月1次）、亲自做首次技术分享、主持关键评审 |
| **时间保障** | 每周三下午（协调会）、每周五下午（分享）不安排其他会议 |
| **强制执行** | 前4周即使有人抱怨也要坚持，形成习惯后就顺畅了 |
| **及时激励** | 对表现好的及时表扬，对不配合的及时沟通 |
| **持续跟进** | 每周你自己过一遍检查清单，确保工作落地 |

## 六、风险与应对

| 风险 | 可能性 | 应对措施 |
|:-----|:------:|:---------|
| 周协调会流于形式 | 中 | 严格控制时间，必须有行动项，你每月参加1次监督 |
| 技术分享无人参加 | 中 | 强制要求、与绩效挂钩、你带头先讲 |
| 试点项目严重延期 | 中 | 每周跟踪进度，及时调配资源，必要时调整范围 |
| 组间协作仍然不畅 | 中 | 主任亲自协调，明确优先级，必要时调整对接人 |
