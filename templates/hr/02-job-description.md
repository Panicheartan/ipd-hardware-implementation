## 岗位体系总览

### 组织架构

```
部门经理
├── 技术团队A（单机研发）
│   ├── 技术专家
│   ├── 高级工程师
│   ├── 工程师
│   └── 助理工程师
├── 技术团队B（模块研发）
│   ├── 技术专家
│   ├── 高级工程师
│   ├── 工程师
│   └── 助理工程师
└── 技术团队C（专项技术）
    ├── 技术专家
    ├── 高级工程师
    ├── 工程师
    └── 助理工程师
```

|------|:----:|----------|------|

## 岗位一：助理工程师（E1）

### 基本信息

| 项目 | 内容 |
|------|------|
| 岗位名称 | 助理工程师 |
| 岗位级别 | E1 |
| 直接上级 | 工程师/高级工程师 |
| 直接下级 | 无 |

### 岗位职责

#### 主要职责

| 序号 | 职责内容 | 权重 |
|:----:|----------|:----:|
| 1 | 在指导下完成简单的电路设计、调试工作 | 30% |
| 2 | 参与原理图绘制、PCB Layout辅助工作 | 25% |
| 3 | 负责样品焊接、测试、记录工作 | 25% |
| 4 | 编写技术文档（测试报告、BOM等） | 15% |
| 5 | 学习并掌握公司研发流程和规范 | 5% |

#### 协作职责

- 配合工程师完成设计任务
- 参与团队技术分享和学习
- 协助完成实验室管理相关工作

### 任职要求

#### 学历与专业

| 项目 | 要求 |
|------|------|
| 学历 | 本科及以上 |
| 专业 | 电子工程、通信工程、自动化等相关专业 |
| 经验 | 应届生或1年以下相关经验 |

#### 知识技能

| 类别 | 要求 |
|------|------|
| 专业知识 | 掌握模拟/数字电路基础理论 |
| 设计工具 | 熟悉Altium Designer/Cadence等EDA工具基本操作 |
| 测试仪器 | 了解示波器、万用表、电源等基本仪器使用 |
| 编程能力 | 了解C语言，能进行简单的嵌入式程序调试 |
| 文档能力 | 能编写基本的技术文档和测试报告 |

#### 素质要求

| 素质项 | 要求 |
|--------|------|
| 学习能力 | 快速学习新技术、新知识 |
| 执行力 | 按时保质完成交办任务 |
| 责任心 | 对工作认真负责，注重细节 |
| 团队协作 | 积极沟通，乐于配合 |

### 绩效考核要点

| 考核维度 | 权重 | 关键指标 |
|----------|:----:|----------|
| 工作质量 | 30% | 设计正确率、文档规范性、测试准确性 |
| 工作效率 | 25% | 任务完成及时率、工作量饱和度 |
| 工作态度 | 25% | 学习积极性、主动性、责任心 |
| 团队协作 | 20% | 沟通配合度、团队贡献度 |

### 职业发展

#### 晋升路径

```
助理工程师（E1）→ 工程师（E2）
```

#### 晋升条件

- 任职满1年
- 能独立完成简单模块设计
- 熟练掌握公司研发流程
- 通过E2能力评估

## 岗位二：工程师（E2）

### 基本信息

| 项目 | 内容 |
|------|------|
| 岗位名称 | 工程师 |
| 岗位级别 | E2 |
| 直接上级 | 高级工程师/技术组长 |
| 直接下级 | 可指导助理工程师 |

### 岗位职责

#### 主要职责

| 序号 | 职责内容 | 权重 |
|:----:|----------|:----:|
| 1 | 独立完成模块级硬件设计（原理图、PCB） | 30% |
| 2 | 负责模块调试、测试、问题解决 | 25% |
| 3 | 参与技术评审，提出设计优化建议 | 15% |
| 4 | 编写设计文档、测试用例、测试报告 | 15% |
| 5 | 指导助理工程师工作 | 10% |
| 6 | 参与新技术预研和学习 | 5% |

#### 协作职责

- 参与跨模块技术协调
- 参与IPD阶段评审
- 协助完成技术积累和知识分享

### 任职要求

#### 学历与经验

| 项目 | 要求 |
|------|------|
| 学历 | 本科及以上 |
| 经验 | 2-4年硬件研发经验 |

#### 知识技能

| 类别 | 要求 |
|------|------|
| 专业知识 | 深入理解模拟/数字电路、电源设计、信号完整性 |
| 设计工具 | 熟练使用EDA工具，能独立完成PCB设计 |
| 测试能力 | 能独立制定测试方案，解决常见硬件问题 |
| 编程能力 | 熟练使用C语言，能进行嵌入式软件开发 |
| 文档能力 | 能编写高质量的技术文档 |

#### 专业方向（可选）

| 方向 | 特殊要求 |
|------|----------|
| 电源设计 | 熟悉开关电源、LDO、电源管理系统设计 |
| 高速数字 | 熟悉高速信号完整性、PCB叠层设计 |
| 模拟电路 | 熟悉放大器、滤波器、ADC/DAC设计 |
| 射频设计 | 熟悉射频电路、天线设计、无线通信 |

### 绩效考核要点

| 考核维度 | 权重 | 关键指标 |
|----------|:----:|----------|
| 技术能力 | 30% | 设计质量、问题解决能力、技术创新 |
| 工作产出 | 30% | 任务完成率、交付质量、文档质量 |
| 团队协作 | 20% | 跨模块配合、知识分享、评审贡献 |
| 学习成长 | 20% | 技能提升、新技术学习、专利/论文 |

### 职业发展

#### 晋升路径

```
                    → 高级工程师（E3）
工程师（E2）→
                    → 技术组长（M2）
```

- 可兼任技术组长，负责小组技术管理

### 任职要求

#### 学历与经验

| 项目 | 要求 |
|------|------|

| 经验 | 5-8年硬件研发经验，有成功项目经验 |

#### 知识技能

| 类别 | 要求 |
|------|------|
| 系统能力 | 能进行系统架构设计，权衡技术方案 |
| 专业深度 | 在某一领域有深入研究和丰富经验 |
| 问题解决 | 能快速定位并解决复杂技术问题 |
| 创新能力 | 能提出创新性解决方案，申请专利 |
| 沟通能力 | 能与客户、跨部门进行有效技术沟通 |

#### 成果要求

| 类别 | 要求 |
|------|------|
| 专利 | 至少有1项授权专利（发明或实用新型） |
| 项目 | 主导完成至少2个量产项目 |
| 论文 | 有技术论文发表优先 |

| 项目贡献 | 25% | 项目成功度、关键交付物质量 |
| 团队贡献 | 25% | 指导效果、知识分享、流程改进 |
| 技术影响力 | 20% | 专利、论文、技术标准、培训 |
