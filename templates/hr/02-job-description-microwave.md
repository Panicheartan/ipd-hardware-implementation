### 2.4 数字工程师

#### 2.4.1 数字工程师岗位体系

| 层级 | 岗位名称 | 方向 | 核心能力要求 |
|:----:|----------|------|--------------|
| E1 | 助理数字工程师 | FPGA/RFSOC | 掌握数字电路基础，熟悉Verilog/VHDL |
| E2 | 数字工程师 | FPGA/RFSOC | 独立开发数字模块，熟悉时序分析 |
| E3 | 高级数字工程师 | FPGA/RFSOC | 系统级设计，高速接口设计 |
| T3 | 数字技术专家 | FPGA/RFSOC | 架构设计，技术规划 |

#### 2.4.2 数字工程师（FPGA方向）核心职责

1. **FPGA架构设计**：根据系统需求设计FPGA架构
2. **逻辑开发**：完成数字信号处理算法实现
3. **接口设计**：实现高速数据接口（ADC/DAC、光纤、以太网等）
4. **时序优化**：确保时序收敛，满足性能要求
5. **测试验证**：编写测试用例，完成功能验证
6. **技术支持**：支持产品开发组的数字模块需求

## 第七章 职业发展路径

### 7.1 技术开发组职业发展路径

```
助理工程师 → 工程师 → 高级工程师 → 技术专家 → 首席专家
    │            │            │             │            │
   1-2年       3-5年        5-8年         8-12年       12年+
```

### 7.2 产品开发组职业发展路径

```
助理产品工程师 → 产品工程师 → 高级产品工程师 → 主任工程师 → 系统专家
       │               │                │               │            │
      1-2年          3-5年            5-8年           8-12年       12年+
```

### 7.3 产品管理组职业发展路径

**技术路线**：
```
助理工程师 → DFX工程师 → 高级DFX工程师 → DFX专家
```

**管理路线**：
```
工程师 → 项目经理 → 部门经理
```

### 7.4 跨组发展

鼓励员工在不同组之间轮岗发展：
- 技术开发组 → 产品开发组：深化系统理解
- 产品开发组 → 技术开发组：强化专业技能
- 各组 → 产品管理组：培养项目管理能力
