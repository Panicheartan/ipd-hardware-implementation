## 第一章 欢迎致辞

### 1.1 欢迎加入

欢迎你加入微波室！我们是公司专注于微波探测与通信产品研发的核心部门，现有成员25人。我们致力于通过IPD（集成产品开发）体系和学习型组织建设，打造高性能、高可靠性的微波产品。

微波室分为三个实体组和一个虚拟综合管理团队：
- **产品管理组**（8人）：负责需求转化、项目管理、DFX监督与评估
- **产品开发组**（4人）：负责探测类（2条线）和通信类产品开发
- **技术开发组**（11人）：负责天线、射频、数字、电源四大类模块开发
- **虚拟综合管理团队**（兼职）：负责档案、安全、6S、宣传、制度建设

### 1.2 团队愿景

成为微波探测与通信领域的技术引领者，打造模块化、平台化的技术体系，实现产品快速开发与持续创新。

### 1.3 团队使命

- **技术领先**：在天线、射频、数字、电源四大领域持续创新
- **产品卓越**：打造性能稳定、质量可靠的探测与通信产品
- **协同高效**：建立产品开发与技术开发的高效协同机制
- **知识传承**：构建学习型组织，实现知识积累与人才成长

## 第三章 行为准则

### 3.1 核心价值观

| 价值观 | 内涵 | 行为要求 |
|--------|------|----------|
| **客户导向** | 以客户需求为中心 | 深入理解客户需求，快速响应客户反馈 |
| **质量第一** | 质量是产品的生命线 | 严守质量标准，不妥协、不侥幸 |
| **协同合作** | 团队成功大于个人成功 | 主动沟通，乐于分享，互相支持 |
| **持续改进** | 追求卓越的永动力 | 不断学习，勇于创新，持续优化 |
| **诚信正直** | 做人做事的底线 | 实事求是，勇于担责，信守承诺 |

### 3.2 日常工作规范

#### 3.2.1 考勤管理

| 项目 | 规定 |
|------|------|
| 工作时间 | 周一至周五 9:00-18:00（弹性时间 ±1小时） |
| 午休时间 | 12:00-13:00 |
| 迟到/早退 | 每月累计不超过3次，每次不超过30分钟 |
| 请假流程 | 提前1天在OA系统提交申请，紧急情况下电话告知直属上级 |
| 加班管理 | 需提前申请，优先安排调休 |

#### 3.2.2 工作纪律

1. **信息保密**
   - 严守公司商业秘密和技术机密
   - 未经批准不得对外泄露项目信息
   - 离职后仍需遵守保密义务

2. **知识产权**
   - 职务发明归公司所有
   - 及时提交技术专利申请
   - 尊重他人知识产权

3. **信息安全**
   - 妥善保管账号密码
   - 不得擅自安装非授权软件
   - 敏感数据不得存储在个人设备

4. **工作场所**
   - 保持工位整洁有序
   - 实验室遵守安全操作规程
   - 公共区域保持安静

### 3.3 沟通协作规范

#### 3.3.1 会议规范

| 会议类型 | 频率 | 时长 | 要求 |
|----------|:----:|:----:|------|
| 团队例会 | 每周 | ≤1小时 | 提前准备，准时参加，有结论有行动项 |
| 项目周会 | 每周 | ≤1.5小时 | 进展汇报、问题讨论、风险识别 |
| 技术评审会 | 按需 | 视情况 | 提前发送材料，有检查表、有问题记录 |
| 临时协调会 | 按需 | ≤30分钟 | 明确议题，快速决策 |

#### 3.3.2 邮件规范

1. **邮件标题**：简明扼要，包含关键词
2. **收件人**：明确主送、抄送对象
3. **正文结构**：背景 → 问题/需求 → 期望行动
4. **附件命名**：项目名称_文档类型_版本_日期
5. **响应时效**：工作日24小时内回复

#### 3.3.3 即时通讯规范

1. 工作时间内保持在线状态
2. 紧急事项优先使用电话
3. 复杂问题转为会议讨论
4. 群组发言注意专业和礼貌

## 第四章 IPD研发流程规范

### 4.1 研发阶段定义

| 阶段 | 目标 | 关键交付件 | 评审点 |
|------|------|------------|--------|
| **概念阶段** | 验证产品机会 | 市场需求文档、概念方案 | CDCP |
| **计划阶段** | 制定详细计划 | 需求规格书、项目计划 | PDCP |
| **开发阶段** | 完成产品设计 | 详细设计文档、原理图、PCB | 设计评审 |
| **验证阶段** | 验证产品功能 | 测试报告、试产报告 | TR评审 |
| **发布阶段** | 产品上市 | 量产准备、发布材料 | ADCP |
| **生命周期** | 持续运营 | 维护记录、改进方案 | 季度评审 |

### 4.3 文档管理规范

#### 4.3.1 文档命名规范

```
[项目编号]_[文档类型]_[文档名称]_[版本]_[日期]
示例：PRJ001_SRS_电源模块需求规格书_V1.0_20240315
```

#### 4.3.2 文档版本管理

| 版本状态 | 版本号规则 | 说明 |
|----------|------------|------|
| 草稿 | V0.x | 编制过程中 |
| 评审中 | V1.0_RCx | Release Candidate |
| 已发布 | V1.0、V2.0 | 正式版本 |
| 已作废 | V1.0_OBS | 被新版本替代 |

#### 4.3.3 文档存储规范

| 文档类型 | 存储位置 | 权限要求 |
|----------|----------|----------|
| 项目文档 | PLM系统/项目文件夹 | 项目组成员 |
| 技术规范 | 技术文档库 | 全体研发 |
| 共享模板 | 模板库 | 全体可读 |
| 机密文档 | 加密存储 | 授权人员 |

### 4.4 变更管理规范

#### 4.4.1 变更分类

| 变更级别 | 定义 | 审批权限 |
|----------|------|----------|
| 一级变更 | 影响项目目标、成本、进度 | CCB/部门经理 |
| 二级变更 | 影响功能、性能、接口 | 项目经理 |
| 三级变更 | 文档更新、优化改进 | 技术负责人 |

#### 4.4.2 变更流程

1. **变更申请**：填写变更请求表(CR)
2. **影响分析**：评估对需求、设计、进度、成本的影响
3. **变更审批**：按级别提交相应审批人
4. **变更实施**：执行变更，更新相关文档
5. **变更验证**：验证变更效果
6. **变更关闭**：归档变更记录

| 类别 | 具体行为 | 惩戒级别 |
|------|----------|:--------:|
| **考勤** | 无故迟到早退、旷工 | 提醒 → 警告 |
| **保密** | 泄露公司机密 | 警告 → 解除合同 |
| **质量** | 故意隐瞒缺陷、数据造假 | 严重警告 → 解除合同 |
| **协作** | 恶意不配合、破坏团队氛围 | 警告 → 严重警告 |
| **诚信** | 虚报工时、冒领物资 | 严重警告 → 解除合同 |
