## 第一章 欢迎致辞

### 1.1 欢迎加入

欢迎你加入我们的硬件研发团队！这是一个专注于单机和模块硬件研发的创新团队，我们致力于通过IPD（集成产品开发）体系，打造高质量、高可靠性的硬件产品。

### 1.2 团队愿景

成为行业内领先的硬件研发团队，以技术创新驱动产品竞争力，以规范管理保障研发质量。

### 1.3 团队使命

- 交付满足客户需求的高质量硬件产品
- 持续技术创新，保持技术领先优势
- 构建高效协同的研发管理体系

## 第三章 行为准则

### 3.1 核心价值观

| 价值观 | 内涵 | 行为要求 |
|--------|------|----------|
| **客户导向** | 以客户需求为中心 | 深入理解客户需求，快速响应客户反馈 |
| **质量第一** | 质量是产品的生命线 | 严守质量标准，不妥协、不侥幸 |
| **协同合作** | 团队成功大于个人成功 | 主动沟通，乐于分享，互相支持 |
| **持续改进** | 追求卓越的永动力 | 不断学习，勇于创新，持续优化 |
| **诚信正直** | 做人做事的底线 | 实事求是，勇于担责，信守承诺 |

### 3.2 日常工作规范

#### 3.2.1 考勤管理

| 项目 | 规定 |
|------|------|
| 工作时间 | 周一至周五 9:00-18:00（弹性时间 ±1小时） |
| 午休时间 | 12:00-13:00 |
| 迟到/早退 | 每月累计不超过3次，每次不超过30分钟 |
| 请假流程 | 提前1天在OA系统提交申请，紧急情况下电话告知直属上级 |
| 加班管理 | 需提前申请，优先安排调休 |

#### 3.2.2 工作纪律

1. **信息保密**
   - 严守公司商业秘密和技术机密
   - 未经批准不得对外泄露项目信息
   - 离职后仍需遵守保密义务

2. **知识产权**
   - 职务发明归公司所有
   - 及时提交技术专利申请
   - 尊重他人知识产权

3. **信息安全**
   - 妥善保管账号密码
   - 不得擅自安装非授权软件
   - 敏感数据不得存储在个人设备

4. **工作场所**
   - 保持工位整洁有序
   - 实验室遵守安全操作规程
   - 公共区域保持安静

### 3.3 沟通协作规范

#### 3.3.1 会议规范

| 会议类型 | 频率 | 时长 | 要求 |
|----------|:----:|:----:|------|
| 团队例会 | 每周 | ≤1小时 | 提前准备，准时参加，有结论有行动项 |
| 项目周会 | 每周 | ≤1.5小时 | 进展汇报、问题讨论、风险识别 |
| 技术评审会 | 按需 | 视情况 | 提前发送材料，有检查表、有问题记录 |
| 临时协调会 | 按需 | ≤30分钟 | 明确议题，快速决策 |

#### 3.3.2 邮件规范

1. **邮件标题**：简明扼要，包含关键词
2. **收件人**：明确主送、抄送对象
3. **正文结构**：背景 → 问题/需求 → 期望行动
4. **附件命名**：项目名称_文档类型_版本_日期
5. **响应时效**：工作日24小时内回复

#### 3.3.3 即时通讯规范

1. 工作时间内保持在线状态
2. 紧急事项优先使用电话
3. 复杂问题转为会议讨论
4. 群组发言注意专业和礼貌

## 第四章 IPD研发流程规范

### 4.1 研发阶段定义

| 阶段 | 目标 | 关键交付件 | 评审点 |
|------|------|------------|--------|
| **概念阶段** | 验证产品机会 | 市场需求文档、概念方案 | CDCP |
| **计划阶段** | 制定详细计划 | 需求规格书、项目计划 | PDCP |
| **开发阶段** | 完成产品设计 | 详细设计文档、原理图、PCB | 设计评审 |
| **验证阶段** | 验证产品功能 | 测试报告、试产报告 | TR评审 |
| **发布阶段** | 产品上市 | 量产准备、发布材料 | ADCP |
| **生命周期** | 持续运营 | 维护记录、改进方案 | 季度评审 |

### 4.2 关键角色职责

#### 4.2.1 PDT（产品开发团队）角色

| 角色 | 主要职责 |
|------|----------|
| **项目经理(LPDT)** | 项目整体管理、跨部门协调、风险控制 |
| **硬件工程师(SE)** | 硬件方案设计、技术攻关、问题解决 |
| **结构工程师** | 结构设计、材料选型、DFM |
| **测试工程师** | 测试策略、测试执行、质量保障 |
| **质量工程师(PQA)** | 流程合规、质量审计、问题跟踪 |
| **采购代表** | 供应商管理、物料采购、成本控制 |
| **制造代表** | 工艺设计、试产支持、量产导入 |

#### 4.2.2 技术评审职责

| 评审角色 | 评审重点 |
|----------|----------|
| 系统工程师 | 系统架构、接口定义、技术方案可行性 |
| 硬件专家 | 电路设计、信号完整性、可靠性设计 |
| 质量代表 | 设计规范、检查项覆盖、风险识别 |
| 制造代表 | DFM、可测试性、成本控制 |

### 4.3 文档管理规范

#### 4.3.1 文档命名规范

```
[项目编号]_[文档类型]_[文档名称]_[版本]_[日期]
示例：PRJ001_SRS_电源模块需求规格书_V1.0_20240315
```

#### 4.3.2 文档版本管理

| 版本状态 | 版本号规则 | 说明 |
|----------|------------|------|
| 草稿 | V0.x | 编制过程中 |
| 评审中 | V1.0_RCx | Release Candidate |
| 已发布 | V1.0、V2.0 | 正式版本 |
| 已作废 | V1.0_OBS | 被新版本替代 |

#### 4.3.3 文档存储规范

| 文档类型 | 存储位置 | 权限要求 |
|----------|----------|----------|
| 项目文档 | PLM系统/项目文件夹 | 项目组成员 |
| 技术规范 | 技术文档库 | 全体研发 |
| 共享模板 | 模板库 | 全体可读 |
| 机密文档 | 加密存储 | 授权人员 |

### 4.4 变更管理规范

#### 4.4.1 变更分类

| 变更级别 | 定义 | 审批权限 |
|----------|------|----------|
| 一级变更 | 影响项目目标、成本、进度 | CCB/部门经理 |
| 二级变更 | 影响功能、性能、接口 | 项目经理 |
| 三级变更 | 文档更新、优化改进 | 技术负责人 |

#### 4.4.2 变更流程

1. **变更申请**：填写变更请求表(CR)
2. **影响分析**：评估对需求、设计、进度、成本的影响
3. **变更审批**：按级别提交相应审批人
4. **变更实施**：执行变更，更新相关文档
5. **变更验证**：验证变更效果
6. **变更关闭**：归档变更记录

## 第五章 培训与发展

### 5.1 培训体系

| 培训类型 | 内容 | 频率 | 对象 |
|----------|------|:----:|------|
| **新员工培训** | 公司文化、制度流程、工具使用 | 入职1周内 | 新员工 |
| **IPD流程培训** | IPD体系、阶段评审、角色职责 | 季度 | 全体研发 |
| **技术培训** | 硬件设计、信号完整性、EMC等 | 月度 | 工程师 |
| **管理培训** | 项目管理、团队管理、沟通技巧 | 半年 | 管理者 |
| **外部培训** | 行业会议、专业认证、技术交流 | 按需 | 骨干员工 |

### 5.2 职业发展通道

#### 5.2.1 技术通道

```
助理工程师 → 工程师 → 高级工程师 → 技术专家 → 首席专家
```

#### 5.2.2 管理通道

```
工程师 → 技术组长 → 项目经理 → 部门经理 → 技术总监
```

### 5.3 技能认证

| 认证方向 | 认证等级 | 要求 |
|----------|----------|------|
| 硬件设计 | 初级/中级/高级/专家 | 通过内部考核+项目经验 |
| IPD流程 | 基础/进阶/专家 | 通过IPD培训+评审实践 |
| 质量意识 | 合格/优秀 | 质量培训+工作表现 |

#### 6.1.2 团队奖励

| 奖项 | 评选标准 | 奖励形式 |
|------|----------|----------|

| 最佳协作奖 | 跨团队协作表现优秀 | 团队奖金+表彰 |

### 6.2 惩戒机制

#### 6.2.1 惩戒分类

| 级别 | 行为 | 惩戒措施 |
|:----:|------|----------|
| **提醒** | 轻微违规、初次犯错 | 口头提醒、记录在案 |
| **警告** | 一般违规、重复犯错 | 书面警告、影响绩效 |
| **严重警告** | 较重违规 | 严重警告、降薪/降职 |
| **解除劳动合同** | 严重违规、触犯法律 | 解除合同、追究责任 |

#### 6.2.2 常见违规行为

| 类别 | 具体行为 | 惩戒级别 |
|------|----------|:--------:|
| **考勤** | 无故迟到早退、旷工 | 提醒 → 警告 |
| **保密** | 泄露公司机密 | 警告 → 解除合同 |
| **质量** | 故意隐瞒缺陷、数据造假 | 严重警告 → 解除合同 |
| **协作** | 恶意不配合、破坏团队氛围 | 警告 → 严重警告 |
| **诚信** | 虚报工时、冒领物资 | 严重警告 → 解除合同 |

## 第八章 沟通与反馈

### 8.1 沟通渠道

| 渠道 | 用途 | 联系方式 |
|------|------|----------|
| 直属上级 | 日常工作、绩效、发展 | 面对面/OA/邮件 |
| 部门经理 | 重大事项、建议反馈 | OA/邮件/面谈 |

| 工会/员工代表 | 权益保障、集体诉求 | 代表姓名/邮箱 |

### 8.2 反馈机制

1. **定期面谈**
   - 试用期面谈：入职1个月、3个月
   - 绩效面谈：每季度一次
   - 发展面谈：每年一次

2. **满意度调查**
   - 季度团队满意度调查
   - 年度员工满意度调查

3. **意见箱/热线**
   - 匿名意见箱
   - 员工热线

### 8.3 申诉机制

如对处理结果有异议，可向上级或人力资源部门申诉，将在5个工作日内给予答复。
