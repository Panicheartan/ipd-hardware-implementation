各团队职责详解

一、产品管理组（偏向项目经理角色）

人员配置：

  • 总人数：8人
  • 组长：1人
  • 研发管理：3人
  • 可靠性管理：4人

可靠性管理4人分工：

岗位 人数 负责领域
可靠性管理 1人 产品可靠性设计、可靠性评估、寿命分析
电磁兼容性管理 1人 EMC设计、EMC测试、电磁兼容评估
元器件管理 1人 元器件选型、优选库管理、元器件质量
环境适应性管理 1人 环境试验、环境适应性设计、防护设计

核心职责：

  1. 对外沟通与需求管理：

    • 对外沟通需求
    • 需求分析
    • 客户需求管理

  2. 项目管理：

    • 项目进度把控
    • 项目DFX监督（Design for X：面向制造、测试、可靠性等的设计）
    • 项目交付

  3. 需求转化：

    • 将需求转化为标准化的硬件需求规格说明

  4. 组织评估（产品完成后）：

    • 可靠性评估
    • 可制造性评估
    • 环境适应性评估
    • 成本评估
    • 电磁兼容性评估

  5. 生产移交与批产支持：

    • 评估通过后，受控全套设计资料
    • 将资料移交生产部门组织批产
    • 协调解决批产过程中的问题

关键输出：

  • 标准化硬件需求规格说明
  • 评估报告（可靠性、可制造性、环境适应性、成本、EMC）
  • 受控设计资料包
  • 项目交付物

二、产品开发组

人员配置：

  • 总人数：4人
  • 组长：1人
  • 产品工程师：3人（分别负责3类产品）

产品工程师分工：

工程师 负责产品 说明
工程师1 微波探测类产品① 探测类产品线1
工程师2 微波探测类产品② 探测类产品线2
工程师3 通信类产品 通信类产品线

计划补充人员（未来）：

  • 结构工程师：支撑产品的结构设计
  • 嵌入式开发工程师：支撑产品的嵌入式软件开发

核心职责：

  1. 单机架构研究：

    • 将单机指标分解为合适的模块指标
    • 选择合适的集成方式
    • 确定模块或组合的性能指标
    • 确定机械接口和电气接口

  2. 集成开发：

    • 根据产品管理组的标准化硬件需求规格说明开展硬件产品开发
    • 选取技术开发组开发的通用模块进行集成
    • 快速形成性能稳定、质量可靠的产品

  3. 技术牵引：

    • 从系统架构上牵引技术开发组的技术发展方向

  4. 方案设计和工程设计：

    • 在结构工程师和嵌入式开发工程师到位后，共同开展方案设计和工程设计

负责的产品类别（3类）：

  • 微波探测类产品（2人负责，2条产品线）
  • 通信类产品（1人负责，1条产品线）

关键输出：

  • 单机架构设计方案
  • 模块指标分解文档
  • 接口定义文档（机械接口、电气接口）
  • 集成后的硬件产品
  • 产品设计资料

三、技术开发组

人员配置：

  • 总人数：11人
  • 组长：1人
  • 各专业工程师：10人

专业分工（四大技术方向）：

专业类别 人数 细分方向 具体工作内容
天线 3人 通信类天线 通信系统天线设计、仿真、测试
探测类无源天线 探测系统无源天线设计
探测类有源天线 探测系统有源天线设计
射频 3人 接收机 接收通道射频电路设计
发射机 发射通道射频电路设计
频率源 频率合成、时钟电路设计
数字 2人 数字信号处理 FPGA、RFSOC等数字电路开发
电源 2人 低压二次电源 为探测、通信单机中的射频、天线、数字电路供电

供电关系说明：

  • 电源组开发的二次电源为其他三组（天线、射频、数字）供电
  • 覆盖探测类和通信类单机

核心职责：

  1. 开发通用模块

    • 开发标准化的通用模块，供产品开发组选用
    • 建立模块库，形成技术积累

  2. 定制开发

    • 根据产品开发组的具体要求，开发定制化的模块或组件

  3. 技术发展推动：

    • 从专业技术发展角度推动技术进步
    • 从元器件技术发展角度推动技术进步
    • 迭代模块组件，使性能不断提高

  4. 支持产品开发：

    • 提供通用模块供产品开发组选取集成
    • 提供技术支持和问题解决方案

  5. 技术预研：

    • 跟踪行业技术发展趋势
    • 开展新技术、新工艺预研
  • 通用模块（天线、射频、数字、电源）
  • 模块性能规格
  • 技术演进路线图

四、虚拟综合管理团队（兼职性质）

组成：

  • 兼职设计师（由其他组成员兼任）
  • 部门领导

性质：

  • 虚拟团队（非固定编制）
  • 成员均为兼职，无专职人员

兼职角色设置：

兼职岗位 职责说明 人员来源
档案管理 部门档案资料管理、归档、借阅管理 各组设计师兼任
安全管理 实验室安全、信息安全、生产安全 各组设计师兼任
6S管理 整理、整顿、清扫、清洁、素养、安全 各组设计师兼任
宣传管理 部门宣传、文化活动、信息发布 各组设计师兼任
制度建设 部门制度编写、修订、宣贯 各组设计师兼任

主要职责：

  1. 跨组协调和资源调配
  2. 重要事项决策支持
  3. 部门日常运营管理（档案、安全、6S、宣传）
  4. 部门制度体系建设与维护
  5. 协助部门领导开展管理工作

团队管理方式

组长设置

  • 各组设立一个组长
  • 组长职责：
    1. 组织好该组的相关工作
    2. 协调组内资源
    3. 协调跨组资源

关键术语和概念

术语 定义
组/团队 同义词，指产品管理组、产品开发组、技术开发组三个实体组织
单机 独立的硬件产品单元
模块 具有特定功能的独立组件
组件 比模块更小的功能单元
标准化硬件需求规格说明 产品管理组输出的标准化需求文档
受控设计资料 经过评审和批准的正式设计文档
批产 批量生产

技术领域

  • 微波技术：核心专业领域
  • 探测技术：产品应用领域1
  • 通信技术：产品应用领域2

接口类型

  • 机械接口：物理连接接口
  • 电气接口：电气信号接口

评估维度

产品管理组组织对完成的产品进行以下评估：

  1. 可靠性评估 - 产品的可靠性和寿命
  2. 可制造性评估 - 生产的可行性和良率
  3. 环境适应性评估 - 适应各种环境条件的能力
  4. 成本评估 - 生产成本和物料成本
